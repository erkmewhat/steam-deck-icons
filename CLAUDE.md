# Stream Deck Sim Racing — Claude Code Instructions

## On Session Start

1. Run `bash tools/setup.sh` to ensure latest code and dependencies
2. Check `.current-game` for active game context (lmu or ace)
3. Review memory files for current project phase and pending work

## Git Workflow

- **Never commit directly to `main`** — it's for tagged releases only
- Work on `develop` or `feature/<name>` branches
- Create feature branches off `develop`: `git checkout -b feature/<name> develop`
- Commit at natural milestones with descriptive messages — don't batch up large changes
- Push after committing so work is accessible from other machines
- When a feature is complete, merge to `develop`
- For releases: cut `release/vX.Y.Z` from develop, tag, merge to `main`
- Current version: pre-v1.0 (fine-tuning phase)

## Adding a New Game

Always follow the 5-step flow:
1. **Ask** — what game and what actions?
2. **Research** — keybindings, telemetry APIs, community standards
3. **Propose** — present action list, icon designs, profile layout
4. **Confirm** — get user approval before writing code
5. **Execute** — create icon pack, plugin actions, profile, deploy config

Use `tools/new-game.py` as the scaffolding tool.

## Build & Deploy

```bash
bash tools/deploy.sh [lmu|ace]    # Full pipeline: build → bundle → install → validate
cd plugin && npm run build         # TypeScript compile only
bash tools/setup.sh                # Sync repo, install deps, verify build
```

Deploy installs to `%APPDATA%/Elgato/StreamDeck/{Plugins,IconPacks,ProfilesV3}/`.

## Key Conventions

- **Actions** are TypeScript classes extending `HotkeyAction` → `SingletonAction`
- Each action = one `.ts` file in `plugin/src/actions/`, compiled to `.js` in the `.sdPlugin/bin/actions/` dir
- **Icons** are SVGs in icon pack `icons/` dirs, synced to plugin `imgs/actions/` by `tools/sync-icons.js`
- **Profiles** are generated by `tools/generate-profile.py` (LMU) or `tools/generate-profile-ace.py` (ACE)
- Manifests (`manifest.json`) in plugin and icon pack dirs define UUIDs and metadata for Stream Deck

## Saving Work

- **Always save research, plans, and decisions to memory files** — never rely on conversation context alone
- Update memory files as work progresses, not just at the end
- Commit and push at every natural milestone
